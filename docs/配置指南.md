# 视频分析MCP服务 - 配置指南

## 概述

本文档详细说明如何配置视频分析 MCP HTTP API 服务，包括环境变量、配置文件、API 密钥设置和部署配置。

## 配置架构

新架构采用**混合配置方式**：
- **环境变量**（`.env`）：存储敏感信息（API 密钥、认证密钥）
- **配置文件**（`config/config.json`）：存储非敏感配置（端口、超时、路径等）

---

## 1. 环境变量配置

### 创建 .env 文件

首次部署时，复制环境变量模板：

```bash
cp .env.example .env
```

### 环境变量说明

#### 1.1 API 服务认证（必需）

```bash
# API 服务认证密钥（自定义，用于 Claude 连接时认证）
API_KEY=your-secure-api-key-here
```

**说明**：
- 这是你自己设置的密钥，用于保护 API 服务
- 建议使用强密码（至少 32 位随机字符）
- Claude 配置时需要在 `Authorization` 头中使用此密钥

**生成示例**：
```bash
# 生成随机 API Key
openssl rand -hex 32
```

#### 1.2 阿里百炼 API Key（必需）

用于语音识别（Qwen-ASR）和图像识别（Qwen-VL）：

```bash
DASHSCOPE_API_KEY=your-dashscope-api-key
```

**说明**：
- 获取 API Key：访问 [阿里云百炼控制台](https://bailian.console.aliyun.com/)
- 一个 API Key 同时用于语音识别和图像识别
- 其他配置（端点、模型等）在配置文件 `config/config.json` 中设置

#### 1.3 阿里云 OSS 配置（必需）

用于上传音频文件到 OSS，供 filetrans 模型访问：

```bash
# OSS Access Key ID
OSS_ACCESS_KEY_ID=LTAI5xxxxxxxxxxxxx

# OSS Access Key Secret
OSS_ACCESS_KEY_SECRET=xxxxxxxxxxxxxxxxxxxxxx

# OSS Endpoint（根据你的 Bucket 所在区域）
OSS_ENDPOINT=oss-cn-beijing.aliyuncs.com

# OSS Bucket 名称
OSS_BUCKET_NAME=your-bucket-name
```

**说明**：
- 获取 Access Key：访问 [阿里云 RAM 控制台](https://ram.console.aliyun.com/)
- Endpoint 根据 Bucket 所在区域选择（如北京：oss-cn-beijing.aliyuncs.com）
- Bucket 需要提前创建，建议设置为公共读或配置正确的访问权限
- 用于 qwen3-asr-flash-filetrans 模型的音频文件上传

## 2. 配置文件

### 创建配置文件

```bash
cp config/config.example.json config/config.json
```

### 配置文件结构

```json
{
  "server": { ... },
  "speech": { ... },
  "vision": { ... },
  "workspace": { ... },
  "download": { ... },
  "processing": { ... }
}
```

### 2.1 服务器配置

```json
{
  "server": {
    "host": "0.0.0.0",
    "port": 8000,
    "timeout": 3600,
    "workers": 1
  }
}
```

**配置说明**：
- `host`: 监听地址（`0.0.0.0` 允许外部访问，`127.0.0.1` 仅本地）
- `port`: 服务端口（默认 8000）
- `timeout`: HTTP 请求超时时间（秒，默认 3600 = 1 小时）
- `workers`: Uvicorn worker 数量（默认 1）

### 2.2 语音识别配置

```json
{
  "speech": {
    "model": "qwen3-asr-flash-filetrans",
    "language": "zh"
  }
}
```

**配置说明**：
- `model`: 使用的模型
  - `qwen3-asr-flash-filetrans`：推荐，支持长音频，句子级时间戳，需要 OSS
  - `qwen3-asr-flash`：同步调用，速度快，但有长度限制
- `language`: 指定音频语种
  - `"zh"`：中文（默认）
  - `"en"`：英语
  - `"ja"`：日语
  - `"ko"`：韩语
  - 更多语种请参考 [Qwen-ASR 文档](./外部模型/阿里百炼-Qwen-ASR.md)

**注意**：使用 `qwen3-asr-flash-filetrans` 模型需要配置 OSS（见环境变量配置）

### 2.3 OSS 配置

```json
{
  "oss": {
    "endpoint": "oss-cn-beijing.aliyuncs.com",
    "bucket_name": "video-mcp"
  }
}
```

**配置说明**：
- `endpoint`: OSS Endpoint（根据 Bucket 所在区域）
  - 北京：`oss-cn-beijing.aliyuncs.com`
  - 上海：`oss-cn-shanghai.aliyuncs.com`
  - 杭州：`oss-cn-hangzhou.aliyuncs.com`
  - 更多区域请参考 [OSS 区域列表](https://help.aliyun.com/document_detail/31837.html)
- `bucket_name`: OSS Bucket 名称（需要提前创建）

**注意**：Access Key 等敏感信息在环境变量中配置，不要写在配置文件中

### 2.4 图像识别配置

```json
{
  "vision": {
    "base_url": "https://dashscope.aliyuncs.com/compatible-mode/v1",
    "model": "qwen3-vl-flash",
    "max_tokens": 2000,
    "temperature": 0.7
  }
}
```

**配置说明**：
- `base_url`: 阿里百炼 API 端点
  - 中国内地：`https://dashscope.aliyuncs.com/compatible-mode/v1`
  - 国际：`https://dashscope-intl.aliyuncs.com/compatible-mode/v1`
  - 美国：`https://dashscope-us.aliyuncs.com/compatible-mode/v1`
- `model`: 使用的视觉模型
  - `qwen3-vl-flash`：推荐，速度快，性能强大
  - `qwen-vl-max`：高性能视觉理解
  - `qwen-vl-plus`：平衡性能和成本
- `max_tokens`: 最大输出 Token 数
- `temperature`: 生成温度（0-1，越高越随机）

### 2.4 工作目录配置

```json
{
  "workspace": {
    "base_dir": "/data/workspaces",
    "auto_cleanup_days": 7,
    "max_workspace_size_gb": 10
  }
}
```

**配置说明**：
- `base_dir`: 工作目录根路径（Docker 容器内路径）
- `auto_cleanup_days`: 自动清理天数（0 = 不自动清理）
- `max_workspace_size_gb`: 单个工作目录最大大小（GB）

### 2.5 下载配置

```json
{
  "download": {
    "default_quality": "best",
    "max_file_size_gb": 5,
    "timeout_seconds": 600,
    "retry_times": 3
  }
}
```

**配置说明**：
- `default_quality`: 默认视频质量（`best`/`720p`/`480p`）
- `max_file_size_gb`: 最大文件大小限制（GB）
- `timeout_seconds`: 下载超时时间（秒）
- `retry_times`: 下载失败重试次数

### 2.6 处理配置

```json
{
  "processing": {
    "audio_format": "wav",
    "audio_sample_rate": 16000,
    "frame_format": "jpg",
    "frame_quality": 90
  }
}
```

**配置说明**：
- `audio_format`: 音频格式（`wav`/`mp3`）
- `audio_sample_rate`: 音频采样率（Hz）
- `frame_format`: 视频帧格式（`jpg`/`png`）
- `frame_quality`: 图片质量（1-100）

## 3. Docker 部署配置

### 3.1 docker-compose.yml

基本配置：

```yaml
version: '3.8'

services:
  video-mcp:
    build: .
    ports:
      - "8000:8000"
    volumes:
      - video-data:/data/workspaces
      - ./config:/app/config:ro
    env_file:
      - .env
    environment:
      - CONFIG_PATH=/app/config/config.json
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 10s
      retries: 3

volumes:
  video-data:
```

### 3.2 自定义端口

修改 `docker-compose.yml` 中的 `ports` 配置：

```yaml
ports:
  - "9000:8000"  # 将本地端口改为 9000
```

### 3.3 自定义 Volume 路径

使用主机路径而非 Docker Volume：

```yaml
volumes:
  - /path/on/host:/data/workspaces
  - ./config:/app/config:ro
```

## 5. 获取 API 密钥

### 5.1 阿里百炼 DashScope API Key

**用途**：语音识别（Qwen-ASR）+ 图像识别（Qwen-VL）

1. 访问 [阿里云百炼控制台](https://bailian.console.aliyun.com/)
2. 开通百炼服务
3. 获取 API Key（DashScope API Key）

**服务特点**：

**语音识别（Qwen-ASR）**：
- 支持多语种识别（中文、英文、日语等 20+ 语种）
- OpenAI 兼容接口
- 支持音频 URL 和 Base64 编码输入
- 支持 ITN（逆文本标准化）
- 识别准确度高

**图像识别（Qwen-VL）**：
- 强大的视觉理解能力
- 支持图像描述、视觉问答、物体定位
- 支持图片 URL 和 Base64 编码输入
- OpenAI 兼容接口
- 多模态理解能力

**定价**：
- 按 Token 计费
- 语音识别：每秒音频 = 25 Token
- 图像识别：按图片大小和输出长度计费
- 具体价格请参考 [阿里云官网](https://help.aliyun.com/zh/model-studio/)

**推荐模型**：
- 语音识别：`qwen3-asr-flash`（同步调用，速度快）
- 图像识别：`qwen3-vl-flash`（速度快，性能强大，推荐）

## 6. 安全配置

### 6.1 保护敏感信息

```bash
# 设置 .env 文件权限（仅所有者可读）
chmod 600 .env

# 确保 .env 不被提交到 Git
echo ".env" >> .gitignore
```

### 6.2 API Key 安全

- ✅ 使用强密码（至少 32 位随机字符）
- ✅ 定期轮换 API Key
- ✅ 不要在日志中记录 API Key
- ❌ 不要将 `.env` 提交到版本控制

### 6.3 网络安全

**本地部署**：
```yaml
# 仅监听 localhost
environment:
  - SERVER_HOST=127.0.0.1
```

**公网部署**：
- 使用 HTTPS（Nginx + Let's Encrypt）
- 配置防火墙规则
- 启用 API Key 认证

### 6.4 HTTPS 配置（云服务器）

使用 Nginx 反向代理：

```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location /mcp {
        proxy_pass http://localhost:8000/mcp;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 3600s;
    }

    location /health {
        proxy_pass http://localhost:8000/health;
    }
}
```

## 8. 故障排查

### 8.1 服务无法启动

**问题**：`docker-compose up` 失败

**解决**：
```bash
# 检查 .env 文件是否存在
ls -la .env

# 检查配置文件语法
cat config/config.json | jq .

# 查看详细错误
docker-compose up
```

### 8.2 认证失败

**问题**：`401 Unauthorized`

**解决**：
- 检查 Claude 配置中的 API Key 是否与 `.env` 中的 `API_KEY` 一致
- 确保 `Authorization` 头格式正确：`Bearer <api-key>`

### 8.3 API 调用失败

**问题**：语音识别或图像识别失败

**解决**：
```bash
# 检查环境变量
docker-compose exec video-mcp env | grep SPEECH
docker-compose exec video-mcp env | grep VISION

# 测试 API 连接
docker-compose exec video-mcp curl https://api.cognitive.microsoft.com
```

### 8.4 超时问题

**问题**：处理长视频时超时

**解决**：
1. 增加 `config.json` 中的 `server.timeout`
2. 增加 Claude 客户端的超时设置
3. 检查网络连接速度
