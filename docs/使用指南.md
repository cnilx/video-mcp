# 视频分析MCP服务 - 使用指南

## 概述

本文档介绍如何使用视频分析 MCP HTTP API 服务，包括部署、配置和典型使用场景。

---

## 快速开始

### 1. 系统要求

- Docker 和 Docker Compose
- 至少 5GB 可用磁盘空间
- 网络连接（用于下载视频和调用 API）

### 2. 部署服务

#### 2.1 克隆项目

```bash
git clone https://github.com/yourusername/video-mcp.git
cd video-mcp
```

#### 2.2 配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件
vim .env
```

填入以下必需配置：
```bash
# API 服务认证密钥（自定义）
API_KEY=your-secure-api-key-here

# 阿里百炼 API Key（语音识别 + 图像识别）
DASHSCOPE_API_KEY=your-dashscope-api-key
```

详细配置说明请参见 [配置指南](./配置指南.md)。

#### 2.3 配置服务（可选）

```bash
# 复制配置文件模板
cp config/config.example.json config/config.json

# 根据需要调整配置
vim config/config.json
```

#### 2.4 启动服务

```bash
# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 验证服务
curl http://localhost:8000/health
```

预期输出：
```json
{
  "status": "healthy",
  "version": "2.0.0"
}
```

### 3. 在 Claude 中配置

编辑 Claude 配置文件：

**Windows**: `%APPDATA%\Claude\claude_desktop_config.json`
**macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
**Linux**: `~/.config/Claude/claude_desktop_config.json`

添加以下配置：

```json
{
  "mcpServers": {
    "video-analyzer": {
      "url": "http://localhost:8000/mcp",
      "headers": {
        "Authorization": "Bearer your-api-key-here"
      }
    }
  }
}
```

**注意**：`Authorization` 中的 API Key 必须与 `.env` 文件中的 `API_KEY` 一致。

### 4. 重启 Claude Desktop

配置完成后，重启 Claude Desktop 使配置生效。

## 使用场景

### 场景 1：分析 YouTube 视频

**用户提问**：
```
请帮我分析这个视频：https://www.youtube.com/watch?v=xxxxx
```

**Claude 的处理流程**：

1. **转录视频语音**
   ```
   调用 transcribe_video(url="https://www.youtube.com/watch?v=xxxxx")
   → 内部自动完成：下载视频 → 提取音频 → 语音识别
   → 返回 task_id + 完整转录文本 + 句子级时间戳 + SRT 内容
   ```

2. **分析内容，确定关键时间点**
   ```
   Claude 分析转录文本，识别重要内容出现的时间点
   例如：[10.5, 45.2, 120.8]
   ```

3. **分析关键帧画面**
   ```
   调用 analyze_video_frames(task_id="xxx", timestamps=[10.5, 45.2, 120.8])
   → 内部自动完成：从工作空间取视频 → 提取帧 → 图像识别
   → 返回每帧的画面分析结果
   ```

4. **生成总结**
   ```
   Claude 基于转录文本和图像分析结果，生成结构化总结
   ```

## 最佳实践

### 1. 视频质量

- ✅ 推荐使用 720p 或更高分辨率
- ✅ 确保音频清晰
- ⚠️ 避免背景噪音过大的视频

### 2. 视频长度

- ✅ 5-30 分钟：最佳处理时长
- ⚠️ 30-60 分钟：处理时间较长
- ❌ 超过 60 分钟：建议分段处理

### 3. 内容类型

**适合分析的视频**：
- 教学视频
- 产品演示
- 会议录像
- 技术分享
- 新闻报道

**不适合的视频**：
- 纯音乐视频（无语音）
- 快速剪辑视频（画面切换过快）
- 低质量录屏（文字模糊）

### 4. 成本控制

- 使用较低视频质量（720p 而非 1080p）
- 减少提取的关键帧数量
- 选择性价比高的 API 服务
- 启用缓存避免重复处理

## 常见问题

### Q1: 支持哪些视频平台？

**A**: 支持 1000+ 平台，包括：
- YouTube
- Bilibili（B站）
- 抖音
- Vimeo
- Twitter
- Facebook
- 以及任何可以通过 URL 访问的视频

### Q2: 处理一个视频需要多长时间？

**A**: 取决于视频长度和网络速度：
- 5 分钟视频：约 2-3 分钟
- 30 分钟视频：约 8-10 分钟
- 60 分钟视频：约 15-20 分钟

### Q3: 如何处理私有视频？

**A**: 目前仅支持公开可访问的视频。如果是私有视频，请：
1. 下载到本地
2. 上传到支持的平台（设为公开或不公开）
3. 使用该链接进行分析

### Q4: 识别准确度如何？

**A**: 取决于视频质量和选择的 API 服务：
- 语音识别：90-95% 准确度（清晰音频）
- 图像识别：85-90% 准确度（清晰画面）
- OCR 文字识别：95%+ 准确度（清晰文字）

### Q5: 如何降低成本？

**A**:
1. 选择经济型 API 服务（阿里云、百度）
2. 使用较低视频质量
3. 减少关键帧提取数量
4. 启用缓存机制
5. 仅在必要时使用图像识别

### Q6: 数据会被保存吗？

**A**:
- 视频文件：保存在 Docker Volume 中
- 自动清理：默认 7 天后删除
- 隐私保护：仅本地存储，不上传到云端
- 手动清理：可以随时删除工作目录

## 故障排查

### 问题 1：视频下载失败

**可能原因**：
- 网络连接问题
- 视频不可用或已删除
- 平台限制

**解决方法**：
```bash
# 查看详细错误日志
docker-compose logs -f

# 测试网络连接
docker-compose exec video-mcp curl -I [视频URL]
```

### 问题 2：语音识别失败

**可能原因**：
- API 密钥无效
- API 配额用尽
- 音频质量太差

**解决方法**：
```bash
# 检查 API 配置
docker-compose exec video-mcp env | grep SPEECH

# 测试 API 连接
# 查看 API 服务状态
```

### 问题 3：处理超时

**可能原因**：
- 视频太长
- 网络速度慢
- 服务器资源不足

**解决方法**：
1. 增加超时时间（`config.json` 中的 `server.timeout`）
2. 使用较低视频质量
3. 分段处理长视频

### 问题 4：认证失败

**可能原因**：
- API Key 不匹配
- Authorization 头格式错误

**解决方法**：
1. 检查 Claude 配置中的 API Key
2. 确保与 `.env` 中的 `API_KEY` 一致
3. 确认格式：`Bearer <api-key>`

## 性能优化

### 1. 网络优化

- 使用 CDN 加速视频下载
- 选择地理位置接近的 API 服务区域
- 启用 HTTP/2

### 2. 存储优化

- 定期清理过期文件
- 使用 SSD 存储
- 配置合理的 Volume 大小

### 3. 并发处理

- 当前版本支持单个请求同步处理
- 未来版本将支持任务队列和并发处理
