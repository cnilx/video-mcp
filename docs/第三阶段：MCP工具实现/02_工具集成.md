# 工具集成

## 概述

视频分析 MCP 服务通过 FastMCP 框架注册了两个核心工具，并以 Streamable HTTP 方式挂载到 FastAPI 应用的 `/mcp` 路径。

## 架构

```
FastAPI (server.py)
├── GET  /health          # 健康检查
└── MOUNT /mcp            # MCP Streamable HTTP
    ├── transcribe_video          # 视频转录工具
    └── analyze_video_frames      # 视频帧分析工具
```

核心文件：
- `src/mcp_app.py` - MCP 工具注册与集成
- `src/server.py` - FastAPI 服务入口，挂载 MCP app

## 工具说明

### transcribe_video

转录视频中的语音内容。

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| url | string | 是 | - | 视频URL（YouTube、B站、抖音等） |
| quality | string | 否 | best | 视频质量：best/high/medium/low |
| language | string | 否 | zh | 语言代码：zh/en/ja/ko 等 |

返回字段：
- `success` - 是否成功
- `task_id` - 任务ID（可用于 analyze_video_frames）
- `text` - 转录文本
- `srt_content` - SRT 字幕内容
- `duration` - 视频时长（秒）
- `video_title` - 视频标题

### analyze_video_frames

分析视频指定时间点的画面内容。

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| timestamps | float[] | 是 | - | 时间戳列表（秒） |
| task_id | string | 否 | - | 已有任务ID，复用已下载的视频 |
| video_path | string | 否 | - | 本地视频文件路径 |
| url | string | 否 | - | 视频URL |
| analysis_type | string | 否 | smart | 分析类型：general/detailed/objects/text/scene/smart |
| custom_prompt | string | 否 | - | 自定义分析提示词 |

视频来源优先级：`task_id` > `video_path` > `url`

返回字段：
- `success` - 是否成功
- `task_id` - 任务ID
- `total_requested` - 请求帧数
- `total_succeeded` - 成功帧数
- `frames[]` - 帧分析结果列表
  - `index` - 帧序号
  - `timestamp` - 时间戳
  - `image_url` - 图片URL
  - `description` - 画面描述

## 使用示例

### MCP 客户端配置

在 MCP 客户端（如 Claude Desktop）中添加配置：

```json
{
  "mcpServers": {
    "video-mcp": {
      "url": "http://localhost:8000/mcp"
    }
  }
}
```

### 示例 1：转录视频

```
请转录这个视频的内容：https://www.bilibili.com/video/BV1xxxxxxxxx
```

MCP 工具调用：
```json
{
  "tool": "transcribe_video",
  "arguments": {
    "url": "https://www.bilibili.com/video/BV1xxxxxxxxx",
    "quality": "best",
    "language": "zh"
  }
}
```

### 示例 2：分析视频帧（使用已有 task_id）

先转录视频获得 task_id，再分析特定时间点的画面：

```
分析这个视频第10秒、30秒和60秒的画面内容
```

MCP 工具调用：
```json
{
  "tool": "analyze_video_frames",
  "arguments": {
    "timestamps": [10.0, 30.0, 60.0],
    "task_id": "ws_20260313_xxxxxx",
    "analysis_type": "smart"
  }
}
```

### 示例 3：直接从 URL 分析帧

```json
{
  "tool": "analyze_video_frames",
  "arguments": {
    "timestamps": [5.0, 15.0, 25.0],
    "url": "https://www.youtube.com/watch?v=xxxxxxxxx",
    "analysis_type": "detailed",
    "custom_prompt": "请描述画面中的人物动作和表情"
  }
}
```

### 示例 4：使用 curl 测试

```bash
# 健康检查
curl http://localhost:8000/health

# 初始化 MCP 会话
curl -X POST http://localhost:8000/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "initialize",
    "params": {
      "protocolVersion": "2025-03-26",
      "capabilities": {},
      "clientInfo": {"name": "test", "version": "1.0"}
    }
  }'

# 列出可用工具
curl -X POST http://localhost:8000/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 2,
    "method": "tools/list"
  }'
```

## 参数验证

所有工具在执行前会进行参数验证：

- `url` 必须以 `http://` 或 `https://` 开头
- `timestamps` 不能为空，且每个值不能为负数
- `quality` 必须是 `best/high/medium/low` 之一
- `analysis_type` 必须是 `general/detailed/objects/text/scene/smart` 之一
- `analyze_video_frames` 必须提供 `task_id`、`video_path` 或 `url` 之一

验证失败时返回 `{"success": false, "error": "错误描述"}`。

## 错误处理

工具执行过程中的错误会被统一捕获并返回结构化的错误信息：

```json
{
  "success": false,
  "error": "具体错误描述"
}
```

错误类型包括：
- 参数验证错误（在工具执行前拦截）
- 业务逻辑错误（如视频下载失败、识别失败）
- 服务内部错误（未预期的异常）

所有工具调用都会记录详细日志，包括调用参数、执行结果和耗时。

## 日志格式

```
[MCP] transcribe_video 调用: url=..., quality=best, language=zh
[MCP] transcribe_video 完成: success=True, task_id=ws_xxx, 耗时=45.2s
```
