# 基础服务架构

## 架构概述

本项目采用 **MCP SDK + FastAPI** 架构，实现可远程部署的视频分析 MCP 服务。

## 核心组件

### 1. FastAPI - HTTP 服务层

**职责**：
- HTTP/HTTPS 服务器
- API 路由管理
- Bearer Token 认证
- 长超时支持（默认 3600 秒）
- 统一错误处理

**API 端点**：
- `GET /health` - 健康检查
- `POST /mcp` - MCP 协议端点

### 2. MCP SDK - 协议层

**职责**：
- MCP 协议实现
- 工具注册和路由
- 参数验证
- 响应格式化

**提供的工具**：
1. `download_video` - 下载在线视频
2. `extract_audio` - 提取音频
3. `transcribe_audio` - 语音转文本
4. `extract_frames` - 提取视频帧
5. `analyze_frames` - 图像识别

## 架构图

```
Claude Desktop/API
        ↓ HTTP/HTTPS + Bearer Token
FastAPI 服务层
  ├─ 认证中间件 (auth.py)
  └─ 路由处理 (server.py)
        ↓
MCP SDK 协议层
  ├─ 协议解析
  ├─ 工具路由
  └─ 参数验证
        ↓
MCP 工具层 (tools/)
  ├─ download.py
  ├─ extract_audio.py
  ├─ transcribe.py
  └─ ...
        ↓
核心处理层 (core/)
  ├─ downloader.py (yt-dlp)
  ├─ audio.py (ffmpeg)
  ├─ speech.py (Qwen-ASR)
  └─ vision.py (Qwen-VL)
```

## 请求处理流程

1. **Claude 发起请求** → POST /mcp + Bearer Token
2. **FastAPI 接收** → 认证验证 → 路由转发
3. **MCP SDK 处理** → 解析协议 → 调用工具
4. **工具层执行** → 调用核心模块 → 返回结果
5. **MCP SDK 格式化** → 封装为 MCP 协议格式
6. **FastAPI 返回** → HTTP 200 + JSON
7. **Claude 接收** → 解析并使用结果

## 为什么选择这个方案

### 与 stdio 方案对比

| 特性 | stdio 方案 | HTTP 方案 |
|------|-----------|----------|
| 部署位置 | 仅本地 | 本地/云服务器 |
| Docker 支持 | ❌ | ✅ |
| 多客户端 | ❌ | ✅ |
| 性能 | 较差 | 优秀 |
| 认证 | ❌ | ✅ |
| 监控 | ❌ | ✅ |

### 选择理由

1. **远程部署** - 支持局域网/云服务器部署
2. **容器化** - Docker 封装所有依赖
3. **高性能** - 服务常驻，避免重复初始化
4. **安全性** - API Key 认证，可配置 HTTPS
5. **可扩展** - 易于添加监控、日志、缓存

## 技术栈

```
应用层: yt-dlp, ffmpeg, Qwen-ASR, Qwen-VL
   ↓
工具层: MCP 工具定义 (tools/)
   ↓
协议层: MCP SDK
   ↓
传输层: FastAPI + uvicorn
   ↓
部署层: Docker + Docker Compose
```

## 启动服务

### 方式 1：使用启动脚本

```bash
# Linux/Mac
./start_server.sh

# Windows
start_server.bat
```

### 方式 2：直接运行

```bash
# 激活虚拟环境
source venv/Scripts/activate  # Linux/Mac
venv\Scripts\activate.bat     # Windows

# 启动服务
python -m uvicorn src.server:app --host 0.0.0.0 --port 8000 --reload
```
