# 配置管理

## 概述

配置管理模块提供了完整的配置加载、验证和热重载功能，支持从配置文件和环境变量读取配置。

## 配置文件

### 配置文件位置

- 默认配置文件：`config/config.json`
- 可通过环境变量 `CONFIG_PATH` 指定自定义路径

### 配置文件结构

```json
{
  "server": {
    "host": "0.0.0.0",
    "port": 8000,
    "timeout": 3600
  },
  "speech": {
    "base_url": "https://dashscope.aliyuncs.com/compatible-mode/v1",
    "model": "qwen3-asr-flash",
    "enable_itn": false,
    "language": null
  },
  "vision": {
    "base_url": "https://dashscope.aliyuncs.com/compatible-mode/v1",
    "model": "qwen3-vl-flash",
    "max_tokens": 2000,
    "temperature": 0.7
  },
  "workspace": {
    "base_dir": "/data/workspaces",
    "auto_cleanup_days": 7
  },
  "download": {
    "default_quality": "best",
    "max_file_size_gb": 5
  }
}
```

## 环境变量

### 必需的环境变量

在项目根目录创建 `.env` 文件：

```bash
# API 服务认证（必需）
API_KEY=your-secure-api-key-here

# 阿里百炼 API Key（必需）- 用于语音识别和图像识别
DASHSCOPE_API_KEY=your-dashscope-api-key
```

### 可选的环境变量

```bash
# 自定义配置文件路径
CONFIG_PATH=config/custom-config.json
```

## 配置项说明

### 服务器配置 (server)

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| host | string | "0.0.0.0" | 服务器监听地址 |
| port | int | 8000 | 服务器端口（1-65535） |
| timeout | int | 3600 | 请求超时时间（秒） |

### 语音识别配置 (speech)

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| base_url | string | "https://dashscope.aliyuncs.com/compatible-mode/v1" | API 基础 URL |
| model | string | "qwen3-asr-flash" | 语音识别模型 |
| enable_itn | bool | false | 是否启用逆文本归一化 |
| language | string | null | 语言代码（如 "zh", "en"） |

### 图像识别配置 (vision)

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| base_url | string | "https://dashscope.aliyuncs.com/compatible-mode/v1" | API 基础 URL |
| model | string | "qwen3-vl-flash" | 图像识别模型 |
| max_tokens | int | 2000 | 最大 token 数 |
| temperature | float | 0.7 | 温度参数（0.0-2.0） |

### 工作空间配置 (workspace)

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| base_dir | string | "/data/workspaces" | 工作目录基础路径 |
| auto_cleanup_days | int | 7 | 自动清理天数 |

### 下载配置 (download)

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| default_quality | string | "best" | 默认视频质量（best/worst/720p/1080p/480p） |
| max_file_size_gb | int | 5 | 最大文件大小（GB） |

## 使用方法

### 基本使用

```python
from src.utils.config import config

# 获取配置项
host = config.server_host
port = config.server_port

# 使用 get 方法获取配置（支持点号路径）
timeout = config.get("server.timeout", 3600)

# 获取敏感信息
api_key = config.api_key
dashscope_key = config.dashscope_api_key
```

### 自定义配置路径

```python
from src.utils.config import Config

# 使用自定义配置文件
custom_config = Config(config_path="path/to/config.json")
```

### 启用配置热重载

```python
from src.utils.config import Config

# 启用自动重载
config = Config(auto_reload=True)

# 配置文件修改后会自动重新加载
# 每次访问配置时会检查文件是否修改
host = config.server_host  # 自动检查并重载
```

### 手动重载配置

```python
from src.utils.config import config

# 手动重新加载配置
success = config.reload()
if success:
    print("配置重新加载成功")
else:
    print("配置重新加载失败")
```

### 检查并重载配置

```python
from src.utils.config import config

# 检查配置文件是否修改，如果修改则重新加载
reloaded = config.check_and_reload()
if reloaded:
    print("检测到配置变化，已重新加载")
```

## 配置验证

配置管理模块使用 Pydantic 进行配置验证，确保配置项的类型和值符合要求。

### 验证规则

1. **端口号验证**：必须在 1-65535 之间
2. **视频质量验证**：只允许 best/worst/720p/1080p/480p
3. **数值范围验证**：
   - timeout >= 1
   - max_tokens >= 1
   - temperature: 0.0-2.0
   - auto_cleanup_days >= 1
   - max_file_size_gb >= 1

### 验证失败处理

- 如果配置文件格式错误或验证失败，将使用默认配置
- 所有错误和警告都会记录到日志中
- 如果缺少必需的环境变量，会记录警告信息

## 配置优先级

1. **环境变量**：敏感信息（API_KEY、DASHSCOPE_API_KEY）
2. **配置文件**：应用配置（config.json）
3. **默认值**：内置默认配置

## 最佳实践

### 1. 分离敏感信息

- 将 API Key 等敏感信息放在 `.env` 文件中
- 不要将 `.env` 文件提交到版本控制系统
- 使用 `.env.example` 作为模板

### 2. 使用配置验证

- 配置管理模块会自动验证配置项
- 确保配置值符合类型和范围要求
- 查看日志了解验证错误

### 3. 配置热重载

- 开发环境可以启用 `auto_reload=True`
- 生产环境建议关闭自动重载，使用手动重载
- 重载配置不会影响正在运行的请求

### 4. 配置文件管理

- 为不同环境创建不同的配置文件
- 使用 `CONFIG_PATH` 环境变量切换配置
- 保持配置文件的 JSON 格式正确

## 示例

### 完整示例

```python
from src.utils.config import Config
from loguru import logger

# 创建配置实例（启用热重载）
config = Config(auto_reload=True)

# 检查必需的环境变量
if not config.api_key:
    logger.error("缺少 API_KEY 环境变量")
    exit(1)

if not config.dashscope_api_key:
    logger.error("缺少 DASHSCOPE_API_KEY 环境变量")
    exit(1)

# 使用配置
logger.info(f"服务器将在 {config.server_host}:{config.server_port} 启动")
logger.info(f"工作目录: {config.workspace_base_dir}")
logger.info(f"语音识别模型: {config.speech_model}")
logger.info(f"图像识别模型: {config.vision_model}")

# 在运行时重新加载配置
if config.reload():
    logger.info("配置已更新")
```

### 在 FastAPI 中使用

```python
from fastapi import FastAPI
from src.utils.config import config

app = FastAPI()

@app.get("/config/reload")
async def reload_config():
    """重新加载配置"""
    success = config.reload()
    return {
        "success": success,
        "message": "配置重新加载成功" if success else "配置重新加载失败"
    }

@app.get("/config/info")
async def get_config_info():
    """获取配置信息"""
    return {
        "server": {
            "host": config.server_host,
            "port": config.server_port,
            "timeout": config.server_timeout
        },
        "workspace": {
            "base_dir": config.workspace_base_dir,
            "auto_cleanup_days": config.workspace_auto_cleanup_days
        }
    }
```

## 故障排查

### 配置文件加载失败

**问题**：配置文件不存在或格式错误

**解决方案**：
1. 检查配置文件路径是否正确
2. 验证 JSON 格式是否正确
3. 查看日志了解具体错误信息
4. 系统会自动使用默认配置

### 环境变量未设置

**问题**：缺少必需的环境变量

**解决方案**：
1. 创建 `.env` 文件
2. 设置 `API_KEY` 和 `DASHSCOPE_API_KEY`
3. 确保 `.env` 文件在项目根目录

### 配置验证失败

**问题**：配置值不符合验证规则

**解决方案**：
1. 检查配置值的类型和范围
2. 参考配置项说明调整配置
3. 查看日志了解验证错误详情

### 配置热重载不生效

**问题**：修改配置文件后未生效

**解决方案**：
1. 确保启用了 `auto_reload=True`
2. 检查配置文件是否保存成功
3. 手动调用 `config.reload()` 重新加载
4. 查看日志确认重载是否成功
