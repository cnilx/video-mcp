# B站视频下载412错误解决方案

## 问题描述

下载B站视频时遇到 HTTP 412 错误：
```
ERROR: [BiliBili] 1x9PWzuEKc: Unable to download webpage: HTTP Error 412: Precondition Failed
```

## 问题原因

**412 Precondition Failed** 错误表示：
1. 请求头不完整或不符合B站服务器要求
2. 缺少必要的浏览器特征标识
3. 触发了B站的反爬虫机制

## 解决方案

### 核心思路

参考抖音下载的成功经验，采用**双重保障策略**：

1. **预解析视频信息**：在下载前先访问B站页面，提取视频标题和元数据
2. **增强HTTP请求头**：使用完整的浏览器请求头，模拟真实浏览器访问
3. **改进yt-dlp配置**：使用更宽松的重试和错误处理策略

### 实现步骤

#### 1. 添加B站URL预解析方法

**文件**: `src/core/downloader.py:317-365`

```python
async def _resolve_bilibili_url(self, url: str) -> Optional[tuple[str, str]]:
    """
    解析B站视频链接，从页面提取播放地址和标题（绕过412错误）
    """
    import httpx
    import json
    import re

    # 使用完整的浏览器请求头
    headers = {
        'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36',
        'Accept': 'text/html,application/xhtml+xml,application/xml;q=0.9,image/avif,image/webp,*/*;q=0.8',
        'Accept-Language': 'zh-CN,zh;q=0.9,en;q=0.8',
        'Accept-Encoding': 'gzip, deflate, br',
        'Referer': 'https://www.bilibili.com/',
        'Connection': 'keep-alive',
        'Upgrade-Insecure-Requests': '1',
        'Sec-Fetch-Dest': 'document',
        'Sec-Fetch-Mode': 'navigate',
        'Sec-Fetch-Site': 'none',
        'Sec-Fetch-User': '?1',
    }

    async with httpx.AsyncClient(follow_redirects=True, timeout=30.0) as client:
        response = await client.get(url, headers=headers)
        html = response.text

        # 从页面提取 __INITIAL_STATE__ JSON 数据
        json_pattern = r'<script>window\.__INITIAL_STATE__=({.*?});'
        matches = re.findall(json_pattern, html)

        if matches:
            data = json.loads(matches[0])
            video_data = data.get('videoData', {})
            title = video_data.get('title', 'bilibili_video')

            # 清理标题中的特殊字符
            title = re.sub(r'[<>:"/\\|?*]', '', title).strip()[:100]

            return (url, title)

    return None
```

## 技术对比

### 抖音 vs B站 解决方案差异

| 特性 | 抖音方案 | B站方案 |
|------|---------|---------|
| **核心策略** | 提取CDN直链 | 预解析+增强配置 |
| **提取内容** | 视频播放URL | 视频标题和元数据 |
| **下载方式** | 直接下载CDN链接 | 使用yt-dlp下载原始URL |
| **绕过机制** | 完全绕过API | 增强请求头通过验证 |
| **User-Agent** | 移动端 | 桌面端 |

### 为什么B站不能完全复制抖音方案？

1. **视频加密**：B站视频有DRM保护，CDN链接需要特殊签名
2. **分段下载**：B站视频通常分段存储，需要合并
3. **yt-dlp支持**：B站的yt-dlp提取器功能完善，只需改进配置即可

## 关键改进点

### 1. 请求头完整性

**之前**：
```python
'Sec-Fetch-Dest': 'empty',      # API请求特征
'Sec-Fetch-Mode': 'cors',       # 跨域请求特征
'Sec-Fetch-Site': 'same-site',  # 同站请求特征
```

**现在**：
```python
'Sec-Fetch-Dest': 'document',   # 文档请求特征
'Sec-Fetch-Mode': 'navigate',   # 导航请求特征
'Sec-Fetch-Site': 'none',       # 直接访问特征
'Sec-Fetch-User': '?1',         # 用户发起特征
```

### 2. 预解析机制

通过预先访问页面：
- ✅ 建立会话状态
- ✅ 获取必要的Cookie
- ✅ 提取视频元数据
- ✅ 验证URL有效性

### 3. 错误处理增强

```python
'extractor_retries': 5,      # 提取器重试5次
'fragment_retries': 10,      # 分片重试10次
'sleep_interval': 1,         # 请求间隔1秒
'max_sleep_interval': 3,     # 最大间隔3秒
```
