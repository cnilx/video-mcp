# 图像识别模块

## 概述

图像识别模块集成阿里百炼 Qwen-VL 模型，提供强大的图像分析和描述生成能力。支持单张和批量图像分析，适用于视频帧分析、图像内容理解等场景。

## 功能特性

- **多种分析类型**：通用描述、详细描述、物体识别、文字识别（OCR）、场景识别
- **灵活的输入方式**：支持本地文件和 URL
- **自动 OSS 上传**：本地图片自动上传到 OSS 获取公网 URL
- **批量处理**：支持并发批量分析多张图片
- **失败重试**：内置重试机制，提高成功率
- **结构化结果**：返回结构化的分析结果，便于后续处理

## 核心类

### VisionAnalyzer

图像识别分析器，负责调用阿里百炼 Qwen-VL API 进行图像分析。

#### 初始化参数

```python
VisionAnalyzer(
    api_key: Optional[str] = None,           # 阿里百炼 API Key
    base_url: str = "https://dashscope.aliyuncs.com/api/v1",  # API 基础 URL
    model: str = "qwen3-vl-flash",           # 视觉模型名称
    max_tokens: int = 2000,                  # 最大 token 数
    temperature: float = 0.7,                # 温度参数
    oss_uploader: Optional[OSSUploader] = None,  # OSS 上传器
    max_retries: int = 3,                    # 失败重试次数
)
```

#### 支持的模型

- `qwen3-vl-plus`：Qwen3-VL 系列中性能最强的模型
- `qwen3-vl-flash`：速度更快，成本更低（推荐）
- `qwen-vl-max`：Qwen2.5-VL 系列中效果最佳的模型
- `qwen-vl-plus`：Qwen2.5-VL 系列，速度更快

### AnalysisType

分析类型枚举：

- `GENERAL`：通用描述
- `DETAILED`：详细描述
- `OBJECTS`：物体识别
- `TEXT`：文字识别（OCR）
- `SCENE`：场景识别

### ImageAnalysisResult

单张图像分析结果：

```python
@dataclass
class ImageAnalysisResult:
    success: bool                    # 是否成功
    image_path: str                  # 本地路径（如果是本地文件）
    image_url: str                   # 图片 URL
    description: str                 # 分析描述
    analysis_type: AnalysisType      # 分析类型
    error: Optional[str]             # 错误信息
    metadata: dict                   # 元数据
```

### BatchAnalysisResult

批量分析结果：

```python
@dataclass
class BatchAnalysisResult:
    success: bool                           # 是否成功
    results: List[ImageAnalysisResult]      # 各图片分析结果
    total_requested: int                    # 请求总数
    total_succeeded: int                    # 成功数量
    total_failed: int                       # 失败数量
    error: Optional[str]                    # 错误信息
```

## 使用示例

### 1. 基础使用

```python
from src.core.vision import VisionAnalyzer, AnalysisType
from src.utils.oss import OSSUploader

# 初始化
analyzer = VisionAnalyzer(
    api_key="your-dashscope-api-key",
    model="qwen3-vl-flash",
    oss_uploader=OSSUploader(),
)

# 分析图片 URL
result = await analyzer.analyze_image(
    image_source="https://example.com/image.jpg",
    analysis_type=AnalysisType.GENERAL,
)

if result.success:
    print(f"图片描述: {result.description}")
else:
    print(f"分析失败: {result.error}")
```

### 2. 分析本地图片

```python
# 本地图片会自动上传到 OSS
result = await analyzer.analyze_image(
    image_source="/path/to/local/image.jpg",
    analysis_type=AnalysisType.DETAILED,
)

print(f"图片 URL: {result.image_url}")
print(f"详细描述: {result.description}")
```

### 3. 自定义提示词

```python
result = await analyzer.analyze_image(
    image_source="https://example.com/image.jpg",
    custom_prompt="请识别图片中的所有文字，并以 JSON 格式输出。",
)

print(result.description)
```

### 4. 批量分析

```python
image_urls = [
    "https://example.com/image1.jpg",
    "https://example.com/image2.jpg",
    "https://example.com/image3.jpg",
]

result = await analyzer.analyze_batch(
    image_sources=image_urls,
    analysis_type=AnalysisType.OBJECTS,
    max_concurrent=3,  # 最大并发数
)

print(f"成功: {result.total_succeeded}/{result.total_requested}")

for i, r in enumerate(result.results):
    if r.success:
        print(f"图片 {i+1}: {r.description}")
```

### 5. 分析视频帧

```python
from src.core.frames import FrameExtractor

# 提取视频帧
extractor = FrameExtractor(oss_uploader=OSSUploader())
frame_result = await extractor.extract_frames(
    video_path="video.mp4",
    output_dir="frames",
    timestamps=[1.0, 5.0, 10.0],
)

# 分析视频帧
if frame_result.success:
    analysis_result = await analyzer.analyze_frames(
        frame_infos=frame_result.frames,
        analysis_type=AnalysisType.SCENE,
        max_concurrent=3,
    )

    print(f"帧分析完成: {analysis_result.total_succeeded}/{analysis_result.total_requested}")
```

## 配置说明

### 环境变量

在 `.env` 文件中配置：

```bash
# 阿里百炼 API Key
DASHSCOPE_API_KEY=sk-xxx

# OSS 配置（用于上传本地图片）
OSS_ACCESS_KEY_ID=your-access-key-id
OSS_ACCESS_KEY_SECRET=your-access-key-secret
OSS_ENDPOINT=oss-cn-hangzhou.aliyuncs.com
OSS_BUCKET_NAME=your-bucket-name
```

### 配置文件

在 `config/config.json` 中配置：

```json
{
  "vision": {
    "base_url": "https://dashscope.aliyuncs.com/api/v1",
    "model": "qwen3-vl-flash",
    "max_tokens": 2000,
    "temperature": 0.7
  }
}
```

## 最佳实践

### 1. 选择合适的模型

- **高精度场景**：使用 `qwen3-vl-plus`，适用于物体定位、文档解析等
- **通用场景**：使用 `qwen3-vl-flash`（推荐），性价比高
- **简单场景**：使用 `qwen-vl-plus`，速度快成本低

### 2. 优化提示词

```python
# 通用描述
"请描述这张图片的内容。"

# 详细描述
"请详细描述这张图片，包括场景、物体、人物、动作、颜色、氛围等所有细节。"

# 物体识别
"请识别并列出图片中的所有物体。"

# 文字识别
"请识别并提取图片中的所有文字内容。"

# 结构化输出
"请识别图片中的物体，并以 JSON 格式输出：[{\"name\": \"物体名称\", \"confidence\": 0.95}]"
```

### 3. 控制并发数

```python
# 根据 API 限流调整并发数
result = await analyzer.analyze_batch(
    image_sources=images,
    max_concurrent=3,  # 建议 2-5
)
```

### 4. 错误处理

```python
result = await analyzer.analyze_image(image_source=url)

if not result.success:
    if "未初始化" in result.error:
        # API Key 未配置
        pass
    elif "上传" in result.error:
        # OSS 上传失败
        pass
    else:
        # API 调用失败
        pass
```

## 性能优化

### 1. 使用 OSS 加速

本地图片会自动上传到 OSS，确保 OSS 配置正确以提高上传速度。

### 2. 批量处理

使用 `analyze_batch` 进行批量处理，利用并发提高效率：

```python
# 不推荐：串行处理
for image in images:
    result = await analyzer.analyze_image(image)

# 推荐：批量并发处理
result = await analyzer.analyze_batch(images, max_concurrent=3)
```

### 3. 重试机制

内置重试机制会自动重试失败的请求，可通过 `max_retries` 参数调整：

```python
analyzer = VisionAnalyzer(
    api_key=api_key,
    max_retries=5,  # 增加重试次数
)
```

## 常见问题

### 1. 本地图片无法分析

**原因**：OSS 未配置或配置错误

**解决**：检查 `.env` 文件中的 OSS 配置，确保 Bucket 有公网访问权限

### 2. API 调用失败

**原因**：API Key 无效、网络问题、限流

**解决**：
- 检查 API Key 是否正确
- 检查网络连接
- 降低并发数，避免触发限流

### 3. 识别结果不准确

**原因**：提示词不够明确、图片质量差

**解决**：
- 使用更详细的提示词
- 提高图片质量和分辨率
- 尝试使用更强大的模型（如 `qwen3-vl-plus`）
