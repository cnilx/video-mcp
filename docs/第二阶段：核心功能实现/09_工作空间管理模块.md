# 09 工作空间管理模块

## 概述

实现视频处理任务的工作空间生命周期管理，为每个任务提供隔离的文件目录，并自动管理磁盘空间。

## 模块文件

| 文件 | 说明 |
|------|------|
| `src/utils/workspace.py` | 工作空间管理器核心实现 |
| `src/utils/config.py` | 配置模型（WorkspaceConfig） |
| `tests/test_workspace.py` | 单元测试（17 个用例） |

## 目录结构

```
base_dir/
├── {workspace_id}/          # 每个任务一个工作空间
│   ├── .created_at          # 创建时间戳文件
│   ├── video/               # 下载的视频文件
│   ├── audio/               # 提取的音频文件
│   ├── frames/              # 提取的视频帧
│   └── output/              # 输出结果（字幕、分析报告等）
└── ...
```

## 核心功能

### WorkspaceManager

| 方法 | 说明 |
|------|------|
| `create(workspace_id)` | 创建工作空间，自动生成 ID（日期+UUID） |
| `get_info(workspace_id)` | 获取工作空间信息（大小、创建时间等） |
| `get_path(workspace_id, subdir)` | 获取工作空间或子目录路径 |
| `list_workspaces()` | 列出所有工作空间，按创建时间排序 |
| `delete(workspace_id)` | 删除指定工作空间 |
| `cleanup_expired()` | 清理超过 `auto_cleanup_days` 天的工作空间 |
| `cleanup_subdir(workspace_id, subdir)` | 清理指定子目录内的所有文件 |
| `get_total_size()` | 获取所有工作空间总大小 |

### 自动管理机制

- 创建新工作空间时自动清理过期空间
- 总大小超过 `max_size_gb` 时自动淘汰最旧的工作空间
- 支持按子目录粒度清理临时文件

## 配置项

```json
{
  "workspace": {
    "base_dir": "/data/workspaces",
    "auto_cleanup_days": 7,
    "max_size_gb": 10.0
  }
}
```

## 使用示例

```python
from src.utils.workspace import WorkspaceManager

manager = WorkspaceManager(
    base_dir="/data/workspaces",
    max_size_gb=10.0,
    auto_cleanup_days=7,
)

# 创建工作空间
ws = manager.create()
video_dir = manager.get_path(ws.workspace_id, "video")

# 清理子目录
manager.cleanup_subdir(ws.workspace_id, "video")

# 删除工作空间
manager.delete(ws.workspace_id)
```
