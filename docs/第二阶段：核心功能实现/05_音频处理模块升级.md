# 音频处理模块升级说明

## 更新日期
2026-03-11

## 更新内容

### 1. 新增 OSS 上传模块

创建了 `src/utils/oss.py` 模块，用于将本地文件上传到阿里云 OSS，获取公网可访问的 URL。

**主要功能：**
- 上传文件到 OSS
- 生成公网访问 URL
- 生成签名 URL（用于私有 Bucket）
- 删除 OSS 文件
- 检查文件是否存在

**配置要求：**
```bash
OSS_ACCESS_KEY_ID=your-oss-access-key-id
OSS_ACCESS_KEY_SECRET=your-oss-access-key-secret
OSS_ENDPOINT=oss-cn-hangzhou.aliyuncs.com
OSS_BUCKET_NAME=your-bucket-name
```

### 2. 升级语音识别模块

将语音识别模型从 `qwen3-asr-flash` 升级为 `qwen3-asr-flash-filetrans`。

**主要变更：**

#### 2.1 模型切换
- **旧模型**: `qwen3-asr-flash` - 支持本地文件，但有长度限制，不支持句子级时间戳
- **新模型**: `qwen3-asr-flash-filetrans` - 支持长音频，提供句子级时间戳，但只支持公网 URL

#### 2.2 处理流程变更

**旧流程（分片处理）：**
```
本地音频文件 → 分片 → 逐片转录 → 合并结果
```

**新流程（不分片）：**
```
本地音频文件 → 上传到 OSS → 获取公网 URL → 提交转录任务 → 等待完成 → 下载结果
```

#### 2.3 新增方法

- `_transcribe_with_filetrans()` - 使用 filetrans 模型转录
- `_submit_filetrans_task()` - 提交 filetrans 转录任务
- `save_srt()` - 保存转录结果为 SRT 字幕文件

#### 2.4 API 变更

`AudioProcessor.__init__()` 新增参数：
```python
def __init__(
    self,
    api_key: Optional[str] = None,
    model: str = "qwen3-asr-flash-filetrans",  # 默认使用新模型
    language: str = "zh",
    oss_uploader: Optional[OSSUploader] = None,  # 新增 OSS 上传器
):
```

`transcribe_audio()` 新增参数：
```python
async def transcribe_audio(
    self,
    audio_path: str,
    auto_split: bool = False,
    segment_duration: Optional[int] = None,
    use_filetrans: bool = True,  # 新增：是否使用 filetrans 模型
) -> TranscriptionResult:
```

### 3. SRT 字幕文件生成

新增 `save_srt()` 方法，可以将转录结果保存为标准 SRT 字幕格式。

**SRT 格式示例：**
```
1
00:00:00,000 --> 00:00:03,500
这是第一句话

2
00:00:03,500 --> 00:00:07,200
这是第二句话
```

**使用方法：**
```python
# 转录音频
result = await processor.transcribe_audio(audio_path)

# 保存 SRT 文件
await processor.save_srt(result, "output.srt")
```

## 使用示例

### 基本使用

```python
import asyncio
from src.core.audio import AudioProcessor
from src.utils.oss import OSSUploader

async def main():
    # 初始化 OSS 上传器
    oss_uploader = OSSUploader()

    # 初始化音频处理器
    processor = AudioProcessor(
        api_key="your-dashscope-api-key",
        model="qwen3-asr-flash-filetrans",
        language="zh",
        oss_uploader=oss_uploader,
    )

    # 转录音频
    result = await processor.transcribe_audio("audio.mp3")

    if result.success:
        print(f"转录文本: {result.text}")
        print(f"句子数量: {len(result.segments)}")

        # 保存 SRT 文件
        await processor.save_srt(result, "audio.srt")

asyncio.run(main())
```

### 使用旧模型（分片处理）

如果不想使用 OSS，可以继续使用旧模型：

```python
processor = AudioProcessor(
    api_key="your-dashscope-api-key",
    model="qwen3-asr-flash",  # 使用旧模型
    language="zh",
)

# 转录音频（自动分片）
result = await processor.transcribe_audio(
    audio_path="audio.mp3",
    use_filetrans=False,  # 不使用 filetrans
    auto_split=True,  # 启用自动分片
)
```

## 配置说明

### 环境变量

在 `.env` 文件中添加以下配置：

```bash
# 阿里百炼 API Key（必需）
DASHSCOPE_API_KEY=your-dashscope-api-key

# 阿里云 OSS 配置（使用 filetrans 模型时必需）
OSS_ACCESS_KEY_ID=your-oss-access-key-id
OSS_ACCESS_KEY_SECRET=your-oss-access-key-secret
OSS_ENDPOINT=oss-cn-hangzhou.aliyuncs.com
OSS_BUCKET_NAME=your-bucket-name
```

### 配置文件

在 `config/config.json` 中添加：

```json
{
  "speech": {
    "model": "qwen3-asr-flash-filetrans",
    "language": "zh"
  },
  "oss": {
    "endpoint": "oss-cn-hangzhou.aliyuncs.com",
    "bucket_name": "your-bucket-name"
  }
}
```

## 优势对比

### 使用 filetrans 模型的优势

| 特性 | qwen3-asr-flash | qwen3-asr-flash-filetrans |
|------|-----------------|---------------------------|
| 音频长度限制 | 有限制（需分片） | 无限制 |
| 句子级时间戳 | 不支持 | 支持 |
| 处理方式 | 同步/分片 | 异步任务 |
| 文件要求 | 本地文件 | 公网 URL |
| SRT 生成 | 基于分段 | 基于句子 |

### 使用 filetrans 模型的劣势

- 需要配置 OSS
- 需要上传文件（增加网络传输时间）
- 异步处理（需要等待任务完成）

## 注意事项

1. **OSS 配置**：使用 filetrans 模型必须配置 OSS
2. **网络要求**：需要稳定的网络连接上传文件
3. **成本**：OSS 存储和流量会产生费用
4. **清理**：建议定期清理 OSS 中的临时文件
5. **安全**：确保 OSS Bucket 的访问权限配置正确

## 故障排查

### 问题 1：OSS 上传失败

**可能原因：**
- OSS 配置错误
- 网络连接问题
- Bucket 权限不足

**解决方法：**
- 检查环境变量配置
- 验证 OSS AccessKey 是否正确
- 确认 Bucket 存在且有写入权限

### 问题 2：转录任务超时

**可能原因：**
- 音频文件过大
- API 服务繁忙

**解决方法：**
- 增加超时时间
- 稍后重试
- 联系阿里云技术支持

### 问题 3：SRT 文件格式错误

**可能原因：**
- 转录结果中没有时间戳
- 使用了不支持时间戳的模型

**解决方法：**
- 确保使用 filetrans 模型
- 检查 API 响应中是否包含时间戳数据
