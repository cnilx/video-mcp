# 视频分析MCP服务 - 设计文档

## 背景和目标

### 需求背景
开发一个MCP服务，为AI助手提供视频分析能力。该服务能够：
1. 下载在线视频（支持多平台）
2. 提取音频并转录为文本
3. 提取指定时间点的视频帧
4. 对视频帧进行图像识别和理解

### 核心价值
- 让AI助手能够"观看"和理解视频内容
- 提供视频的原始分析数据（转录文本、图像识别结果）
- AI助手基于这些数据自主进行分析和总结

## 技术选型

### 编程语言
**Python** - 理由：
- AI/ML工具生态丰富（yt-dlp、ffmpeg-python等）
- MCP Python SDK支持完善
- 异步处理能力强
- FastAPI 提供高性能 HTTP API 支持

### 核心依赖
- **FastAPI**: 高性能 Web 框架，提供 HTTP/SSE 支持
- **mcp**: MCP Python SDK（HTTP transport）
- **yt-dlp**: 视频下载（支持1000+网站）
- **ffmpeg-python**: 音频/视频处理
- **httpx/aiohttp**: 异步HTTP客户端
- **Pillow**: 图像处理
- **pydantic**: 数据验证和配置管理
- **uvicorn**: ASGI 服务器

### 外部服务
- **语音识别**: 阿里百炼 Qwen-ASR（OpenAI 兼容接口）
- **图像识别**: 阿里百炼 Qwen-VL（OpenAI 兼容接口）

### 部署方式
- **Docker**: 容器化部署，包含所有依赖（Python、ffmpeg等）
- **Docker Compose**: 简化部署配置
- **Docker Volume**: 持久化工作目录和配置文件

### 架构分层

**HTTP 服务层（server.py + auth.py）**
- 职责：HTTP/SSE 服务、MCP 协议处理、API 认证
- 特点：使用 FastAPI 提供高性能异步服务
- 通信：支持 MCP over HTTP/SSE

**工具层（tools/）**
- 职责：MCP工具接口定义，参数验证，结果格式化
- 特点：薄封装，专注于接口适配
- 保持：与原设计相同的 5 个工具接口

**核心层（core/）**
- 职责：实际处理逻辑，可独立测试
- 特点：模块化设计，每个模块职责单一
- 保持：与原设计相同的处理逻辑

**工具层（utils/）**
- 职责：通用功能支持
- 特点：工作目录管理、配置管理等基础设施
- 保持：与原设计相同的工具函数

## MCP工具接口设计

### 1. download_video - 下载视频
```python
输入：
- url: str - 视频链接

输出：
{
  "video_path": "/path/to/video.mp4",
  "title": "视频标题",
  "duration": 300.5,
  "format": "mp4",
  "workspace_id": "uuid"
}
```

**说明**：视频质量使用配置文件中的 `download.default_quality` 设置。

### 2. extract_audio - 提取音频
```python
输入：
- video_path: str - 视频文件路径

输出：
{
  "audio_path": "/path/to/audio.wav",
  "duration": 300.5,
  "sample_rate": 16000,
  "channels": 1
}
```

**说明**：音频格式使用配置文件中的 `processing.audio_format` 设置。

### 3. transcribe_audio - 语音转文本
```python
输入：
- audio_path: str - 音频文件路径

输出：
{
  "text": "完整转录文本",
  "segments": [
    {"start": 0.0, "end": 5.2, "text": "这是第一句话"},
    ...
  ],
  "language": "zh"
}
```

**说明**：语言识别使用配置文件中的 `speech.language` 设置（null 为自动检测）。

### 4. extract_frames - 提取视频帧
```python
输入：
- video_path: str - 视频文件路径
- timestamps: list[float] - 时间戳列表（秒）

输出：
{
  "frames": [
    {
      "timestamp": 10.5,
      "path": "/path/to/frame_10.5.jpg",
      "width": 1920,
      "height": 1080
    },
    ...
  ]
}
```

**说明**：图片格式使用配置文件中的 `processing.frame_format` 设置。

### 5. analyze_frames - 图像识别
```python
输入：
- frame_paths: list[str] - 图片路径列表
- prompt: str = "请提取图片中的所有信息内容，包括：文字、标题、正文、图表数据、代码片段、公式、表格等。" - 分析提示词

输出：
{
  "analyses": [
    {
      "frame_path": "/data/workspaces/uuid/frames/frame_10.5.jpg",
      "timestamp": 10.5,
      "content": "Qwen-VL 提取的完整信息内容"
    },
    ...
  ]
}
```

**说明**：
- 使用阿里百炼 Qwen-VL 多模态模型
- 专注于信息提取：文字、图表、代码、PPT内容等
- 支持自定义提示词引导分析方向（适配不同内容类型）
- 返回提取的信息内容（由 Qwen-VL 自由格式输出）

## 核心模块设计

### 0. HTTP 服务和认证（server.py + auth.py）

**HTTP 服务（server.py）**：
- 使用 FastAPI 提供 HTTP/SSE 服务
- 实现 MCP 协议的 HTTP transport
- 支持同步处理（长超时）
- 健康检查端点：`GET /health`
- MCP 端点：`POST /mcp` (HTTP) 或 `GET /mcp/sse` (SSE)

**API 认证（auth.py）**：
- Bearer Token 认证方式
- API Key 通过环境变量 `API_KEY` 配置
- 请求头格式：`Authorization: Bearer <api_key>`
- 认证失败返回 401 Unauthorized

**配置方式**：
- 敏感信息（API keys）：环境变量（`.env` 文件）
- 非敏感配置：配置文件（`config/config.json`）
- Docker 部署时通过 docker-compose.yml 注入环境变量

### 1. 工作目录管理（workspace.py）
**功能**：
- 为每个视频创建独立的工作目录（基于UUID）
- 自动管理临时文件的生命周期
- 提供清理机制（手动清理或自动过期清理）

**目录结构**（Docker Volume 挂载）：
```
/data/workspaces/          # Docker Volume 挂载点
  ├── {uuid1}/
  │   ├── video.mp4
  │   ├── audio.wav
  │   ├── frames/
  │   │   ├── frame_10.5.jpg
  │   │   └── frame_25.3.jpg
  │   └── metadata.json
  └── {uuid2}/
```

### 2. 配置管理（config.py）

**环境变量（敏感信息）**：

# 阿里百炼 API Key（语音识别 + 图像识别）

### 3. 下载器（downloader.py）
- 使用yt-dlp下载视频
- 支持多平台（YouTube、B站、抖音等）
- 支持质量选择和格式转换
- 提供下载进度回调

### 4. 音频处理（audio.py）
- 使用ffmpeg提取音频
- 转换为适合语音识别的格式（16kHz, mono, wav）
- 支持音频片段提取

### 5. 语音识别客户端（speech.py）
- 使用阿里百炼 Qwen-ASR 模型
- OpenAI 兼容接口调用
- 支持多语种识别（20+ 语种）
- 返回带时间戳的分段文本
- 支持 ITN（逆文本标准化）

### 6. 图像识别客户端（vision.py）
- 使用阿里百炼 Qwen-VL 模型
- OpenAI 兼容接口调用
- 支持图像描述、视觉问答、物体定位
- 多模态理解能力
- 支持图片 URL 和 Base64 编码

## 典型使用流程

```
1. Claude 通过 HTTP 调用 MCP 工具 download_video(url="https://youtube.com/...")
   → HTTP POST /mcp (带 Authorization header)
   → 返回 {video_path, workspace_id, duration, ...}

2. Claude 调用 extract_audio(video_path=...)
   → HTTP POST /mcp
   → 返回 {audio_path, duration, ...}

3. Claude 调用 transcribe_audio(audio_path=...)
   → HTTP POST /mcp (同步等待，可能需要几分钟)
   → 返回 {text, segments: [{start, end, text}, ...]}

4. Claude 分析转录文本，决定需要查看的关键时间点
   → 例如：[10.5, 45.2, 120.8]

5. Claude 调用 extract_frames(video_path=..., timestamps=[10.5, 45.2, 120.8])
   → HTTP POST /mcp
   → 返回 {frames: [{timestamp, path}, ...]}

6. Claude 调用 analyze_frames(frame_paths=[...], prompt="请详细描述画面内容")
   → HTTP POST /mcp
   → 返回 {analyses: [{description, detected_text, scene_type, objects, people_count}, ...]}

7. Claude 基于转录文本和图像分析结果，自主进行总结和分析
```

**注意**：
- 所有文件路径都是容器内路径（如 `/data/workspaces/{uuid}/video.mp4`）
- 文件仅用于工具链内部传递，Claude 不直接访问文件
- 最终分析结果以文本形式返回给 Claude

## 错误处理策略

### HTTP 层错误
- 认证失败：返回 401 Unauthorized
- 请求格式错误：返回 400 Bad Request
- 服务器错误：返回 500 Internal Server Error
- 超时：配置足够长的超时时间（默认 3600 秒）

### 下载失败
- 网络错误：重试3次，指数退避
- 不支持的平台：返回明确错误信息
- 视频不可用：返回详细原因

### 处理超时
- 长视频处理：同步等待，HTTP 超时设置为 1 小时
- 超时限制：可通过配置文件调整
- 客户端需要设置相应的超时时间

### API调用失败
- API密钥无效：返回配置错误提示
- 配额超限：返回明确的限流信息
- 服务不可用：自动降级或重试

### 文件系统错误
- 磁盘空间不足：提前检查并警告
- 权限问题：返回明确的权限错误
- 工作目录损坏：自动清理和重建

## 性能优化

- 多帧图像识别可以并发调用API

### 缓存机制

## Docker 部署

### Dockerfile
```dockerfile
FROM python:3.11-slim

# 安装系统依赖
RUN apt-get update && apt-get install -y \
    ffmpeg \
    && rm -rf /var/lib/apt/lists/*

# 设置工作目录
WORKDIR /app

# 复制依赖文件
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制源代码
COPY src/ ./src/
COPY config/ ./config/

# 创建数据目录
RUN mkdir -p /data/workspaces

# 暴露端口
EXPOSE 8000

# 启动服务
CMD ["uvicorn", "src.server:app", "--host", "0.0.0.0", "--port", "8000"]
```

### docker-compose.yml
```yaml
version: '3.8'

services:
  video-mcp:
    build: .
    ports:
      - "8000:8000"
    volumes:
      - video-data:/data/workspaces
      - ./config:/app/config:ro
    env_file:
      - .env
    environment:
      - CONFIG_PATH=/app/config/config.json
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 10s
      retries: 3

volumes:
  video-data:
```
