# 视频分析MCP服务 - 项目概览

## 项目信息

**项目名称**: 视频分析MCP服务（HTTP API 版本）
**版本**: 2.1.0（音频处理升级版）
**创建日期**: 2024-03-10
**更新日期**: 2026-03-11
**技术栈**: Python 3.11+, FastAPI, MCP SDK (HTTP), Docker, yt-dlp, ffmpeg, 阿里云 OSS

## 项目目标

开发一个基于 HTTP API 的 MCP 服务，让 AI 助手能够"观看"和理解视频内容，实现：
1. 多平台视频下载
2. 音频提取和语音识别
3. 基于语义的智能抽帧
4. 图像识别和内容理解
5. 综合分析和总结

## 架构特点

### API 服务模式
- **通信方式**：HTTP/SSE（替代原 stdio 方式）
- **部署方式**：Docker 容器化部署
- **访问方式**：通过 URL 配置，支持本地/局域网/云服务器
- **认证方式**：API Key（Bearer Token）

### 核心优势
- **灵活部署**：可部署在任何支持 Docker 的环境
- **易于扩展**：未来可添加 Web 管理界面
- **简化配置**：Claude 只需配置 URL 和 API Key
- **独立运行**：服务独立于 Claude，可被多个客户端使用

## 核心功能

### 1. 视频下载
- 支持1000+视频平台（YouTube、B站、抖音等）
- 使用yt-dlp实现
- 支持质量选择

### 2. 音频处理
- 使用 ffmpeg 提取音频
- 自动上传到阿里云 OSS
- 支持多种音频格式输出

### 3. 语音识别
- 使用阿里百炼 qwen3-asr-flash-filetrans 模型
- 支持任意长度音频（无需分片）
- 返回句子级毫秒精度时间戳
- 自动生成 SRT 字幕文件
- 自动语言检测

### 4. 智能抽帧
- 基于音频转录内容
- AI模型分析并指定关键时间点
- 精确提取视频帧

### 5. 图像识别
- 专门API识别（Google Vision、百度AI等）
- 视觉语言模型深度理解（GPT-4V、Claude等）
- 组合分析模式

## 技术架构

### 整体架构

```
Claude Desktop/API Client
    ↓ (HTTP/SSE + API Key)
┌─────────────────────────────────────┐
│   MCP HTTP Server (FastAPI)        │
│   - API 认证                         │
│   - HTTP/SSE Transport              │
│   - MCP Protocol Handler            │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│   MCP Tools Layer (5个工具)         │
│   - download_video                  │
│   - extract_audio                   │
│   - transcribe_audio                │
│   - extract_frames                  │
│   - analyze_frames                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│   Core Processing Layer             │
│   - downloader.py (yt-dlp)          │
│   - audio.py (ffmpeg + filetrans)   │
│   - oss.py (阿里云 OSS)              │
│   - vision.py (图像识别API)         │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│   Utils Layer                       │
│   - workspace.py (文件管理)         │
│   - config.py (配置管理)            │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│   Docker Volume (持久化存储)        │
│   /data/workspaces/                 │
└─────────────────────────────────────┘
```

### 数据流

```
视频URL
  ↓ (HTTP POST /mcp + Auth)
下载视频 → 视频文件 (容器内路径)
  ↓
提取音频 → 音频文件
  ↓
上传 OSS → 公网 URL
  ↓
语音识别 → 转录文本（句子级时间戳）+ SRT 字幕
  ↓
AI分析 → 关键时间点列表
  ↓
提取帧 → 图像文件列表
  ↓
图像识别 → 图像分析结果
  ↓
综合总结 → 最终报告（返回给 Claude）
```

## MCP工具接口

| 工具名称 | 功能 | 输入 | 输出 |
|---------|------|------|------|
| download_video | 下载视频 | url, quality | video_path, metadata |
| extract_audio | 提取音频 | video_path | audio_path, metadata |
| transcribe_audio | 语音转文本 | audio_path | text, segments, srt_file |
| extract_frames | 提取视频帧 | video_path, timestamps | frames[] |
| analyze_frames | 图像识别 | frame_paths[], prompt | analyses[] |

## 配置要求

### API 服务配置

**环境变量（敏感信息）**：
- `API_KEY`: 服务认证密钥（自定义）
- `DASHSCOPE_API_KEY`: 阿里百炼 API 密钥（语音识别 + 图像识别）
- `OSS_ACCESS_KEY_ID`: 阿里云 OSS Access Key ID
- `OSS_ACCESS_KEY_SECRET`: 阿里云 OSS Access Key Secret
- `OSS_ENDPOINT`: OSS Endpoint（如 oss-cn-beijing.aliyuncs.com）
- `OSS_BUCKET_NAME`: OSS Bucket 名称

**配置文件（非敏感配置）**：
- 服务器配置（端口、超时时间）
- 语音识别配置（模型：qwen3-asr-flash-filetrans）
- OSS 配置（endpoint、bucket_name）
- 图像识别配置（模型、端点、温度等）
- 工作目录配置
- 下载配置（质量、大小限制）

### 系统依赖

- Docker 和 Docker Compose
- 5GB+ 磁盘空间（用于 Docker Volume）

### 部署环境

支持以下部署方式：
1. **本地部署**：与 Claude Desktop 在同一台机器
2. **局域网部署**：部署在局域网服务器
3. **云服务器部署**：部署在公网服务器（建议配置 HTTPS）

## 成本估算

### 典型10分钟视频

| 服务 | 费用 |
|------|------|
| 语音识别 | $0.10-0.30 |
| 图像识别（5帧） | $0.01-0.05 |
| **总计** | **$0.15-0.35** |

注：不包含Claude自身的使用费用（由用户的Claude订阅覆盖）

### 优化建议

- 使用较低视频质量（720p）
- 减少提取帧数
- 选择性价比高的服务
- 启用缓存机制

## 性能指标

### 处理时间（预估）

| 视频长度 | 下载 | 转录 | 抽帧 | 识别 | 总计 |
|---------|------|------|------|------|------|
| 5分钟 | 30s | 1min | 10s | 30s | ~2.5min |
| 30分钟 | 2min | 5min | 30s | 1min | ~9min |
| 60分钟 | 5min | 10min | 1min | 2min | ~18min |

*实际时间取决于网络速度、API响应时间等因素*
*不包含Claude分析和总结的时间*

### 资源占用

| 资源 | 占用 |
|------|------|
| 磁盘空间 | 视频大小 + 音频大小 + 图片大小 |
| 内存 | ~500MB-1GB |
| CPU | 中等（主要用于ffmpeg处理） |
| 网络 | 视频下载 + API调用 |

## 安全和隐私

### 数据处理

- 所有文件存储在本地工作目录
- 自动清理过期文件（默认7天）
- 不上传原始视频到云端

### API密钥管理

- 配置文件本地存储
- 不包含在版本控制中
- 建议使用环境变量

### 隐私保护

- 仅处理用户指定的视频
- 不收集用户数据
- API调用遵循各服务商隐私政策
